# Workspace Templates

Templates for generating workspace files when opening a new ticket workspace.

## CLAUDE.md

This file is automatically read by Claude Code when the cwd is inside the workspace.

```markdown
# {TICKET-KEY}: {summary}

## Ticket Context
- **Key**: {TICKET-KEY}
- **Type**: {type}
- **Status**: {status}
- **Jira**: https://{domain}/browse/{TICKET-KEY}

## Workspace Files
- `work-log.md` -- 작업 일지 (날짜별 기록)
- `status.md` -- 진행 상태 체크리스트
- `repo/` -- git 저장소

## Available Skills
- `/ticket-workspace:tw-log add "내용"` -- 작업 로그 추가
- `/ticket-workspace:tw-status sync` -- 상태 동기화
- `/ticket-workspace:tw-transition` -- Jira 상태 전환
- `/ticket-workspace:tw-close` -- 워크스페이스 닫기
```

## .tw-config.json

```json
{
  "ticket": "{TICKET-KEY}",
  "summary": "{ticket summary from Jira}",
  "type": "{issue type}",
  "status": "{current Jira status}",
  "repo": "{git repo URL or empty}",
  "branch": "{feature branch name or empty}",
  "baseBranch": "{base branch name or empty}",
  "createdAt": "{ISO-8601 timestamp with timezone}",
  "workspaceStatus": "active"
}
```

## README.md

```markdown
# {TICKET-KEY}: {summary}

| Field    | Value                                           |
|----------|-------------------------------------------------|
| Key      | {TICKET-KEY}                                    |
| Type     | {type}                                          |
| Status   | {status}                                        |
| Priority | {priority}                                      |
| Assignee | {assignee}                                      |
| Sprint   | {sprint or "--"}                                |
| URL      | [{TICKET-KEY}](https://{domain}/browse/{TICKET-KEY}) |

## Description

{description from Jira, preserve markdown formatting}

---
> Auto-generated by ticket-workspace. Last synced: {YYYY-MM-DD HH:mm}
```

## work-log.md

```markdown
# Work Log: {TICKET-KEY}

> {summary}

---

## {YYYY-MM-DD}

### {HH:mm} - Workspace created
- Workspace initialized
- Repository cloned: {repo-name or "skipped"}
- Branch: {branch-name or "N/A"}
```

| Branch        | {branch or "--"} |
